1) Creation du depot git dans le repertoire du travail du projet
	git status
	git add
	git commit
2) Se déplacé entre les commits
	git checkout sha du commit
	git log -> Afficher les commits
3) On peut renommer les sha de commit en tag, car il est plus facile de se souvenir d'un
nom de tag que le sha du commit, 
	Pour le faire on doit se deplacé vers le commit et faire
	git tag <nom_du_tag>

4) Crée une branche et se deplacé vers cette branch
	pour crée une branch il faut d'abord se déplacé vers un commit specifique 
	git branch <nom_du_branch>
Important ! si on fait un commit sur une branch spécifique, cette commit ne sera pas visible dans 
le brach master en tant qu'on a pas faire la fusion avec git merge

5) Pour fusionner notre travail avec le brach master
	Il faut se placer dans le branch master à l'aide de la commade switch <nom_du_branch>
	Et faire la commande git merge

6) Pour savoir sur quel commit ou branch nous nous situons, il faut toujours voir oú se place le HEAD

7) Il faut savoir que si on se deplace sur une branch git specifique, on a accés uniqument qu'aux fichiers lié à cette branche, tout le reste ne nous sera pas accessible même en dehors de git

--------------------GitHub-------------------------
Cloner un dépot sur github: git clone <lien_du_depot.git>

remote: le remote est un pointer ou un connecteur qui permet de faire le lien entre un depot
local et un depot distant

voir le connecter: git remote -v
information complementeur sur le remote: git remote show <nom_du_remote> qui est souvent origin

crée un remote: git remote add <nom_du_remote> suivi du lien distant sur github

--------Pour pousser un branch ou un tag local vers gitHub----------

git push origin <nom_du_branch>

git push origin <not_du_tag>
git puch origin --tags: pour pousser plusieurs tags e même temps

-----------Faire pointer une branch local vers une branch distant-----------
si j'ai un branch local test1 que je voudrais pousser vers le une brranch distant du même nom, je peux me deplacé sur ce branch est faire git push

mais si j'ai une branch local test1 que je veux pousser vers une branch distante appelé master alors il faudra bien précisé le nom comme suit:

pour paramétré les suivi entre branch local est distante: git push --set-upstream origin <nom_du_branch>
git push origin branch_locale:branch_distante
